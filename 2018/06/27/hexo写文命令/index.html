hexo写文命令

发表于 2018-06-27 | 分类于 hexo

每次部署的步骤，可按以下三步来进行。
$ hexo clean
$ hexo generate
$ hexo deploy

1. new 新建文章的指令
$ hexo new [layout] <title>
如果title包含空格的话，请使用引号括起来。
layout为可选项，默认使用_config.yml中的default_layout。

2. generate 生成静态页面至public目录
$ hexo generate
可选参数：
-d ,–deploy 文件生成后立即部署网站
-w , –watch 监视文件变动
该指令可以简写为hexo g

3. deploy发布到网站，将.deploy_git目录部署到GitHub
$ hexo deploy
这里就是发布到_config.yml中deploy中设置的网址上。
参数
-g , –generate 部署前生成静态文件
该命令可以简写hexo d

4. publish发表草稿
$ hexo publish [layout]

5. server启动服务器
$ hexo server
默认情况下，访问网址为：http://localhost:4000/

选项 描述
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日记记录，使用覆盖记录格式

6. clean清除缓存文件 (db.json) 和已生成的静态文件 (public)
$ hexo clean
在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。
